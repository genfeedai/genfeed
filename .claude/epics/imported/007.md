## Summary

Lock/unlock nodes to skip during execution and use cached outputs instead. Supports both individual nodes and groups.

## Key Deliverables

- Lock/unlock individual nodes
- Create/manage node groups
- Skip locked nodes during execution
- Use cached outputs for locked nodes

## Use Cases

- Lock expensive AI generation nodes after getting good results to avoid re-running them
- Group related nodes together for organizational clarity
- Batch lock/unlock operations on groups
